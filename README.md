# Cours_DEP_SI

Bienvenue dans ce dépôt de suivi pour le **DEP en soutien informatique**. Il sert à regrouper les notes, documents, démarches et suivis importants liés au parcours de formation et à la recherche de stage.

## Contenu principal

| Section | Description |
| --- | --- |
| [`stage.md`](./stage.md) | Suivi des démarches de stage : pistes finies, pistes encore ouvertes, relances et contacts à vérifier. |
| Cours | Espace prévu pour organiser les notes, exercices et ressources du DEP. |
| Documents | Espace prévu pour les fichiers utiles : CV, modèles, références et préparation. |

## Suivi de stage

Le fichier le plus important pour l'instant est :

<p align="center">
  <a href="./stage.md">
    <img src="https://img.shields.io/badge/Ouvrir%20le%20suivi-stage.md-14b8a6?style=for-the-badge&logo=github" alt="Ouvrir stage.md" />
  </a>
</p>

Il permet de voir rapidement :

- ce qui est **fini** : refus, pistes fermées, démarches terminées;
- ce qui n'est **pas fini** : candidatures en attente, relances à faire, contacts à clarifier;
- les prochaines dates importantes pour relancer sans trop insister.

## Objectif

Garder un suivi propre et utile pour avancer sans perdre le fil :

```text
Chercher -> Envoyer -> Relancer -> Classer -> Continuer
```

## Repères utiles

| Type | À noter |
| --- | --- |
| Relances | Attendre environ une semaine quand une candidature est récente. |
| Refus | Ne pas relancer sauf nouvelle information claire. |
| Contacts à vérifier | Confirmer si la candidature a déjà été envoyée avant d'agir. |
| Priorités | Garder l'énergie sur les pistes encore ouvertes et pertinentes. |
